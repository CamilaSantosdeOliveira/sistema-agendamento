🚨 ERR_CONNECTION_REFUSED

Problema de conexão com o servidor local

🔧 Configuração de Porta

Porta do Servidor: 8080

URL de Acesso: http://localhost:8080

URL do Sistema: http://localhost:8080/sistema_final.html

❌ Problema Identificado

Erro: "localhost recusou estabelecer ligação"

Causa: O servidor Apache do XAMPP não está rodando na porta 8080

✅ Solução

Você precisa iniciar o servidor Apache no XAMPP Control Panel

1Abrir o XAMPP Control Panel

Existem várias formas de abrir o XAMPP:

  • Menu Iniciar: Digite "XAMPP" e clique no resultado
  • Área de Trabalho: Procure pelo ícone do XAMPP
  • Pasta de Instalação: Vá para C:\xampp\xampp-control.exe
📁 Localize e abra o XAMPP Control Panel

2Iniciar o Apache

No XAMPP Control Panel:

  1. Procure pela linha "Apache"
  2. Clique no botão "Start" ao lado de Apache
  3. Aguarde até o status ficar verde
⚠️ Atenção: Se aparecer um erro sobre porta ocupada, pode ser que outro programa esteja usando a porta 8080

3Iniciar o MySQL

Após o Apache estar rodando:

  1. Procure pela linha "MySQL"
  2. Clique no botão "Start" ao lado de MySQL
  3. Aguarde até o status ficar verde

4Testar a Conexão

Após iniciar ambos os serviços:

  1. Abra seu navegador
  2. Digite: http://localhost:8080
  3. Deve aparecer a página do XAMPP
✅ Sucesso! Se você vê a página do XAMPP, o servidor está funcionando

🔍 Problemas Comuns e Soluções

❌ Porta 8080 ocupada

Sintoma: Apache não inicia, erro de porta

Solução:

  • Feche outros servidores web que possam estar usando a porta 8080
  • Ou mude a porta do Apache para 8081

❌ Acesso Negado

Sintoma: Erro de permissão

Solução:

  • Execute o XAMPP como Administrador
  • Clique com botão direito → "Executar como administrador"

❌ Firewall bloqueando

Sintoma: Serviços iniciam mas não acessa

Solução:

  • Permita o XAMPP no firewall do Windows
  • Ou desative temporariamente o firewall

📞 Precisa de Ajuda?

Se ainda não conseguir resolver:

  • Verifique se o XAMPP está instalado corretamente
  • Reinicie o computador e tente novamente
  • Desinstale e reinstale o XAMPP
  • Consulte a documentação oficial do XAMPP
