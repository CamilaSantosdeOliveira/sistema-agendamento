🔧 SOLUÇÃO DEFINITIVA - MySQL XAMPP

Guia completo para resolver o erro "MySQL shutdown unexpectedly"

❌ PROBLEMA IDENTIFICADO

Erro: MySQL shutdown unexpectedly

Causas possíveis:

  • Porta 3306 bloqueada ou em uso
  • Dependências ausentes
  • Privilégios inadequados
  • Crash do sistema
  • Desligamento por outro método

📋 SOLUÇÕES MANUAIS PASSO A PASSO

🔄 SOLUÇÃO 1: Reiniciar XAMPP

Feche completamente o XAMPP Control Panel
Abra o Gerenciador de Tarefas (Ctrl+Shift+Esc)
Procure e encerre todos os processos relacionados ao MySQL
Abra o XAMPP Control Panel como administrador
Tente iniciar o MySQL novamente

🔧 SOLUÇÃO 2: Verificar Configuração

No XAMPP Control Panel, clique em Config ao lado do MySQL
Selecione my.ini
Verifique se a porta está configurada como 3306
Verifique se o caminho dos dados está correto

Configuração esperada no my.ini:

[mysqld]
port=3306
datadir=C:/xampp/mysql/data
socket=mysql
max_allowed_packet=16M

🚫 SOLUÇÃO 3: Conflito de Porta

Abra o CMD como administrador
Execute: netstat -an | findstr :3306
Se a porta estiver em uso, execute: netstat -ano | findstr :3306
Identifique o PID do processo conflitante
Encerre o processo: taskkill /PID [PID] /F

🗂️ SOLUÇÃO 4: Verificar Diretório de Dados

Vá para C:\xampp\mysql\data
Verifique se existe o arquivo ibdata1
Se não existir, pode haver corrupção de dados
Faça backup e reinstale o MySQL

🔄 SOLUÇÃO 5: Reinstalar MySQL (Último Recurso)

Faça backup da pasta C:\xampp\mysql\data
Delete a pasta C:\xampp\mysql
Baixe o XAMPP novamente
Extraia apenas a pasta mysql
Restaure seus dados

💻 COMANDOS ÚTEIS

Execute estes comandos no CMD como administrador:

# Verificar porta 3306
netstat -an | findstr :3306

# Verificar processos MySQL
tasklist | findstr mysql

# Verificar serviços
sc query mysql

# Encerrar processo por PID (substitua XXXX pelo PID)
taskkill /PID XXXX /F

# Verificar arquivos de log
type C:\xampp\mysql\data\mysql_error.log

📁 ARQUIVOS DE LOG IMPORTANTES

Verifique estes arquivos para mais detalhes:

  • XAMPP MySQL Log: C:\xampp\mysql\data\mysql_error.log
  • XAMPP Control Log: C:\xampp\xampp-control.log
  • Windows Event Viewer: Aplicativos → MySQL

  1. Execute o diagnóstico automático primeiro

✅ DICA IMPORTANTE

Na maioria dos casos, o problema é resolvido com a SOLUÇÃO 1 (reiniciar XAMPP). Se não funcionar, tente as outras soluções na ordem apresentada.
