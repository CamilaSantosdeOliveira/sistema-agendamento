🔧 SOLUÇÃO DEFINITIVA

MySQL Shutdown Unexpectedly

Guia completo para resolver o problema mais comum do XAMPP

❌ PROBLEMA IDENTIFICADO

Erro: "MySQL shutdown unexpectedly"

Duração: 00:00:38

Causa: Arquivos de log corrompidos ou configuração inadequada

🔍 CAUSAS MAIS COMUNS

  • Arquivos de log corrompidos: ib_logfile0, ib_logfile1, ibdata1
  • Configuração inadequada: my.ini com parâmetros incorretos
  • Porta ocupada: Outro processo usando a porta 3306
  • Permissões: Falta de privilégios de administrador
  • Conflito de versões: Múltiplas instalações MySQL
  • Recursos insuficientes: Memória ou espaço em disco

📝 SOLUÇÃO MANUAL PASSO A PASSO

Encerrar Processos MySQL

Abra o CMD como administrador e execute:

taskkill /F /IM mysqld.exe
taskkill /F /IM mysql.exe
taskkill /F /IM mysqld-nt.exe

Verificar Porta 3306

Verifique se a porta está livre:

netstat -an | findstr :3306

Se aparecer algo, aguarde ou encerre o processo que está usando a porta.

Fazer Backup do my.ini

Copie o arquivo de configuração:

copy "C:\xampp\mysql\bin\my.ini" "C:\xampp\mysql\bin\my.ini.backup"

Remover Arquivos de Log Corrompidos

Delete os arquivos problemáticos:

del "C:\xampp\mysql\data\ib_logfile0"
del "C:\xampp\mysql\data\ib_logfile1"
del "C:\xampp\mysql\data\ibdata1"
del "C:\xampp\mysql\data\mysql-bin.index"

Corrigir Arquivo my.ini

Abra C:\xampp\mysql\bin\my.ini e adicione na seção [mysqld]:

innodb_force_recovery=1
max_allowed_packet=16M
skip-grant-tables
innodb_buffer_pool_size=16M
innodb_log_file_size=5M
innodb_log_buffer_size=8M

Iniciar MySQL

Abra o XAMPP Control Panel como administrador e clique em "Start" ao lado do MySQL.

🔧 CONFIGURAÇÃO DE EMERGÊNCIA

⚠️ Se nada funcionar, use esta configuração completa:

Substitua todo o conteúdo do arquivo my.ini por:

[mysqld]
# Configuração de emergência para MySQL shutdown unexpectedly
innodb_force_recovery=1
max_allowed_packet=16M
skip-grant-tables
skip-networking
skip-name-resolve
innodb_buffer_pool_size=16M
innodb_log_file_size=5M
innodb_log_buffer_size=8M
innodb_flush_log_at_trx_commit=2
innodb_lock_wait_timeout=50
port=3306
socket=mysql
key_buffer_size=16M
max_connections=100
table_open_cache=64
thread_cache_size=8
query_cache_size=16M
tmp_table_size=16M
max_heap_table_size=16M

[mysql]
default-character-set=utf8

[client]
default-character-set=utf8
port=3306
socket=mysql

✅ VERIFICAÇÃO

Como verificar se funcionou:

  1. Abra o XAMPP Control Panel
  2. Verifique se o MySQL está com luz verde
  3. Teste o phpMyAdmin: http://localhost/phpmyadmin
  4. Verifique a porta: netstat -an | findstr :3306

📞 SUPORTE

Se ainda não funcionar:
